## 1. Definitions

"GitCheck" refers to the reference implementation of GitCheck, gitcheck.py
as shipped in the lfs-tools suite, updcheck section.

"knowngit" refers to knowngit.txt, a manually maintained file mostly containing
information about the known Git repositories.

"currentgit" refers to lfsqol-git.txt (referred to as currentver.txt in the
GitCheck usage text), an autogenerated file containing the names of Git
repositories and the commit IDs that are packaged in SLFS.

"directive" refers to the three-letter identifier in front of every line in
both knowngit and currentgit.

"parameter" refers to anything in a line that goes after the directive.

## 2. Directives accepted in both knowngit and currentgit

### 2.1. rem
#### Full name: REMark
This directive specifies a comment which should not get printed. Any parameters
specified in the rem directive get thrown out during parsing.
#### Syntax
rem \<arbitrary number of parameters\>

## 3. Directives accepted only in knowngit

### 3.1. ntf
#### Full name: NoTiFication
This directive specifies a comment which should get printed when an update for
the specified package is detected.
#### Syntax
ntf \<repository\> \<beep\> \<author\> \<arbitrary number of parameters\>  
Parameter 1 (repository): Specifies the package name.  
Format: string (package name)  
Parameter 2 (beep): Specifies whether this notification should issue the ASCII  
"Bell" escape sequence to ring the bell on the terminal when printed.  
Format: either y (beep) or n (no beep)  
Parameter 3 (author): Specifies the author of the notification.  
Format: string (arbitrary)  
Parameters 4+ (comment): Specifies the notification itself.  
Format: arbitrary  

### 3.2. rpo
#### Full name: RePO
This directive specifies a source repository.
#### Syntax
rpo \<name\> \<repo\>  
Parameter 1 (name): The name of the package.  
Format: string (package name)  
Parameter 2 (repo): The URL of the Git repository.  
Format: URL (in the format of https://gitstorage.net/repo)  

## 4. Directives accepted only in currentgit

### 4.1. cmt
#### Full name: CoMmiT
This directive specifies a commit of a package currently in SLFS.
#### Syntax
cmt \<name\> \<commit\>  
Parameter 1 (name): The name of the package.  
Format: string (package name)  
Parameter 2 (commit): The commit packaged in SLFS.  
Format: Git commit ID
